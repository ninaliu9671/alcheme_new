# Alcheme · 产品项目书

**alchemy + me · 你是人生的炼金师，这里照见你的魔法**

*版本 v1.3 · 2026年5月 · 阶段：概念验证，寻找种子用户*

---

## 一、项目背景

### 1.1 创始人洞察

Alcheme 诞生于一个非常真实的个人困境：

> 「我知道我做到了，但我不允许自己相信它。」

很多人在外部看起来还不错——有成绩、有认可、有进展——但内心深处始终有一个声音在说「这不算」、「这是运气」、「还不够好」。这不是记忆力的问题，不是缺乏成就，而是一种深层的自我信任危机。

这类人不会主动寻求心理支持，但又被效率工具和打卡应用完全忽视——那些工具解决的是「做更多事」，而不是「相信自己做到的事」。

Alcheme 想填补这个空白。

### 1.2 核心信念

> 你不需要被帮助，你只需要一个见证者。
>
> Alcheme 不创造任何东西。它只是帮你看见，你本来就有的那些。

产品的底层逻辑：

- 外部认可可以触动人，但建立不起真正的自信
- 自我记录容易陷入「这也不算什么」的否定循环
- 真正有效的，是一个有记忆的见证者——它记得你说过什么，然后在某个你没预期的时刻，把那些话还给你

用户用久了，依赖的不是 Alcheme，而是重新建立起了看见自己的能力。有一天她不需要 Alcheme 了——这不是产品失败，这是产品成功。

### 1.3 名字释义

| | 含义 |
|---|---|
| **Alchemy** | 炼金术——将普通的东西提炼成珍贵之物的能力 |
| **Me** | 你自己——你才是自己的炼金师，Alcheme 是你的见证者 |

## 二、产品理念

### 2.1 核心洞察：不是输在事实，而是输在叙事

很多人的问题，不是真的做得不够好，而是**讲给自己听的那个故事版本出了问题**。

同样一件事：
- 事实版本：「我一个人带起了两支队伍，两个产品都做出来了。」
- 内心叙事：「我只是侥幸撑下来了，早晚会被看穿。」

事实没有变，但叙事决定了这个人相信自己是谁。

这不是性格缺陷，也不是矫情。这是人类大脑在特定文化和成长环境下形成的认知习惯——尤其在东亚的成长语境里，自我怀疑往往是一种长期内化的底色，很多人甚至不知道自己有这个问题，只是隐约觉得「不管做了什么，都好像不算数」。

Alcheme 想做的，是帮这些人**重新学会讲自己的故事**。

### 2.2 理论基础：叙事重建

Alcheme 的底层逻辑，与心理学中的**叙事疗法（Narrative Therapy）**高度吻合。

叙事疗法的核心主张是：人不是问题本身，问题是问题。每个人都在用一套内化的「主线故事」理解自己——「我是那种做不下去的人」「我总是不如别人」。这些主线故事会选择性地过滤现实，让所有不符合它的证据都变得「不算数」。

治疗师的工作，是帮人找到那些被主线故事遮蔽掉的**替代证据**，然后用这些证据重建一个更真实的自我叙事。

Alcheme 的 AI 提炼，做的正是这件事：把用户自己说出来的事实，用一个她不会主动用的叙事框架还给她。不是虚假鼓励，是**把被忽视的事实重新变得可见**。

### 2.3 产品定位：培养能力，而非提供治疗

Alcheme **不是心理治疗工具**，也不想让用户觉得自己「需要被治疗」。

真正需要临床支持的人，通常已经有途径接触到专业资源。但还有更大的一个群体——他们功能正常，外部表现不错，但内心始终有一个声音在说「还不够」。他们不会主动寻求心理帮助，因为他们觉得自己「还没到那个程度」。

这个群体，被现有产品完全忽视了：效率工具嫌他们太软，疗愈产品嫌他们太轻。

Alcheme 的入口，不是「你需要帮助吗」，而是「**你想不想更了解自己**」。前者预设了缺陷，后者是一种好奇心的邀请。但最终到达的地方，是一样的。

产品的专业性是藏起来的。用户感受到的，只是被一个有记忆的朋友看见。

### 2.4 终极目标：让用户不再需要 Alcheme

Alcheme 想帮用户培养的，是一种**自我叙事能力**——能够用更真实、更完整的视角看待自己正在经历的事。

这个能力不是天生的，但可以被训练。产品不会把用户划分成不同阶段、推送不同功能。而是**同一个产品，感知到用户在哪里，自然给出对应的回应**——用户不需要知道这条弧线的存在，她只是在某一天，突然发现自己开始相信自己了。

这条成长弧线大致是：

- **被看见**：AI 帮用户看见她看不见的东西，用户接收，感受到被见证
- **开始练习**：AI 在提炼之后轻轻邀请——「现在，你愿意用自己的话说说今天吗？」——用户开始尝试用新的框架描述自己的经历
- **内化**：用户输入里已经开始自己给自己肯定了，AI 退到见证和记录的位置，偶尔发一封信说：「我注意到，你说话的方式和三个月前不一样了。」

> 有一天她不需要 Alcheme 了——这不是产品失败，这是产品成功。

## 三、目标用户

### 3.1 核心画像

Alcheme 的核心用户，是那些「外部看起来还行，但私下不相信自己」的人。

| 她们是谁 | 她们的痛 |
|---|---|
| 有一定成就，但总觉得「不算」 | 明明做到了，但大脑自动找「但是」 |
| 收到夸奖会本能反驳 | 效率工具太冷，疗愈产品太重 |
| 不缺认可，缺的是相信认可的能力 | 没有产品是真正为她设计的 |
| 不会主动说自己需要心理支持 | 复盘期——做了，但看不见价值 |

### 3.2 与现有产品的差异

| | Flomo / 笔记软件 | **Alcheme** |
|---|---|---|
| 输入姿态 | 记录——已想清楚的东西 | 倾诉——还没想清楚的情绪 |
| AI 做什么 | 整理和连接信息 | 把你的话翻译成你能相信的版本 |
| 产品目标 | 让你更聪明、更有条理 | 让你更相信自己 |
| 用完感受 | 更清晰 | **有点想哭** |

## 四、产品功能设计

### 4.1 核心体验

> 用户只需要做一件事——随手说。今天发生了什么，遇到什么，哪怕一句话。
> AI 读懂它，提炼出「我做到的事」，还给你。

### 4.2 三大核心模块

#### 模块一：说说（倾诉入口）

- 用户通过语音或文字输入一段话，不限主题，不要求结构，像在写日记
- 触发动画：炼金师执笔 → 手记本上字迹蹦出
- AI 提炼出 2-4 个点，全部以第一人称「我」开头——「我坚持了……」「我在最难的时候……」——帮用户找到自己本来就有但没说出口的叙事
- 提炼结果呈现后，页面底部有一行轻柔的邀请：

  > 「想不想用自己的声音，留住这几句话？」

  这个设计有明确的心理学依据：在叙事疗法的实践中，咨询师有时会在说完某段话后，邀请来访者把这段话复述一遍——不是重复信息，而是让那些话经由自己的声音发出来，完成一次真正的内化。听别人说和自己说，是两件事。

  在 Alcheme 里，这个邀请同样成立：哪怕用户只是把 AI 提炼的话念出来，也已经是一次有力量的动作。不需要重新创作，不需要比 AI 说得更好——只是把那些话，变成自己的声音。

  具体形式上，用户可以：
  - 录一段语音，念出 AI 提炼的内容，或者用自己的语言重新说一遍
  - 用文字写下来，可以照着念，也可以改写
  - 直接跳过

  无论哪种形式，这段内容都作为「第二次叙述」存入原声记录，与 AI 提炼版本并列保存，互不替换。此环节没有 AI 回应，没有对话，只是一个安静的空间——让用户和这些话，单独待一会儿。

- 用户确认后，一键收录进手记本

#### 模块二：手记本（积累空间）

- 每篇手记 = 一次倾诉的提炼结果，积累成一本书
- 手记按时间排列，用户可以随时翻回去看
- 每篇手记可点开查看 AI 提炼内容，也可选择查看自己的原始输入，以及第二次叙述（如有）
- 口吻：手记内容以第一人称「我」呈现——用户翻看时，读到的是自己的声音，是自己说出来的故事

#### 模块三：信箱（主动来信）

- 系统每天后台静默运行，读取用户历史手记，判断有没有值得写信的时机
- 触发条件不固定，用户无法预期——制造「不期而遇」的惊喜感
- 所有来信存入信箱，按时间累积，用户打开时永远不知道今天有没有新信

### 4.3 来信类型（六种）

| 类型 | 触发逻辑 | 核心感受 |
|---|---|---|
| **深挖同一件事** | 用户反复谈论同一件事，但每次视角不同 | 「你知道你一直在想这件事吗」 |
| **模式发现** | 多条不同记录，背后指向同一个特质 | 「我在你身上看见了一条隐藏的线」 |
| **随机回望** | 系统随机抽取旧记录，冷不丁寄回来 | 「我今天突然想起你说过的这句话」 |
| **近期回顾** | 最近记录积累到一定量，不定期总结 | 「最近这段时间，你经历了这些」 |
| **久别来信** | 沉默超过 N 天未登录 | 「你不在，但我一直记得你」 |
| **悄悄变了** | AI 检测到用户的叙事方式出现了可见的转变——开始主动用更完整、更肯定的视角描述自己 | 「我注意到，你说话的方式和以前不一样了」 |

**关于触发频率：** 所有类型均不设固定周期。系统判断「现在有没有值得写信的东西」，有则写，无则等。用户永远猜不到下一封信什么时候来——这种不确定性，本身就是产品体验的一部分。

**关于「悄悄变了」：** 这封信是整条成长弧线里最有力量的时刻。它不是奖励，不是打卡，是一面镜子——让用户看见自己在变。触发门槛要高，宁可晚发，不能滥发。

### 4.4 口吻设计

**手记：第一人称「我」**

AI 提炼内容全部以「我」开头——「我坚持了三十天」「我一个人带起了两支队伍」。用户翻看手记时，读到的是自己的声音，是她自己说出来的故事，而不是被别人贴上的标签。这种第一人称的力量在于：这句话是从她嘴里出来的，更容易被她自己相信。

**来信：第二人称「你」**

Alcheme 写给用户的信，始终用「你」。来信是一个有记忆的见证者在说话，它记得她说过什么，在某个她没预期的时刻把那些话还给她。「你」在这里制造的是被看见的感觉——有另一双眼睛在，一直都在。

两个空间，两种人称：手记是「我的故事」，来信是「被看见的时刻」。

## 五、产品态度与边界

### 刻意不做的事

| 不做 | 原因 |
|---|---|
| 对话式 AI 交流 | 防止沉迷，保持克制；Alcheme 是见证者，不是陪伴者 |
| 打卡 / 连续天数 | 自我怀疑的人不需要更多压力，需要的是接纳 |
| 社区 / 分享功能 | 这是私密空间，不是表演舞台 |
| 游戏化升级系统 | 升级感会让产品变成效率工具，失去疗愈内核 |
| 广告 | 会摧毁「纯净私密空间」的产品体验 |
| 用户分级功能推送 | 产品通过感知用户状态自然调整回应，而不是把用户贴标签 |

### 产品的克制是卖点

> 你说完，我读完，我记住。不需要你一直在线。
>
> 这种克制，是对那些已经被各种 AI 聊累了的人，最好的礼物。

## 六、技术实现路径

### 6.1 核心技术组成

- **前端** — 手机端 Web App（优先）或微信小程序，移动优先设计
- **AI 接口** — Deepseek V4，负责即时提炼和来信生成
- **数据库** — Supabase，存储用户手记、信箱内容
- **后台 Agent** — 定时任务（每日凌晨），读取历史记录，判断是否触发来信，生成并存入信箱

### 6.2 来信 Agent 逻辑

每天凌晨，系统对每个用户执行：

1. 读取该用户所有历史手记的摘要，包括 AI 提炼版本和用户原声（含第二次叙述）
2. 将摘要传入 Claude，角色设定为「读完所有记录的见证者」
3. Claude 判断：现在有没有值得写信的时机？属于哪种类型？
4. 有则生成对应类型的信，存入用户信箱；无则跳过，明天再判断

判断「悄悄变了」时，Claude 需对比用户早期和近期的叙事方式，识别是否出现了主动的正向叙事转变。触发门槛要高，避免过早或轻率发送。

这个机制实现「不期而遇」感的关键：系统在悄悄做事，用户不知道触发条件。

### 6.3 推送策略

- **短期（MVP）** — 站内信箱，用户打开 App 时看到新信
- **中期** — 邮件推送（保留「收到一封信」的仪式感）
- **长期** — 微信小程序订阅消息（用户主动订阅后可推送）

*注：微信主动推送权限受限，暂不作为核心依赖。*
